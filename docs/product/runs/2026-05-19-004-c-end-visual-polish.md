## 用户角色与权限

- 未登录读者：进入 C 端阅读公开内容，关注首屏可信度、文章查找和详情阅读。
- 回访读者：通过分类、标签或列表继续浏览相近内容，关注路径清晰和扫描效率。
- 内容运营/站点维护者：通过最终视觉效果判断内容站是否专业可信；本需求不涉及后台权限改动。
- 权限边界：C 端公开页面为主，不新增登录态、受保护路由或权限逻辑。

## 核心阅读流程

- 首页进入：识别站点主题 -> 浏览主推荐/最新内容 -> 进入列表或详情。
- 列表浏览：扫描标题、摘要、分类、时间 -> 按分类/标签筛选 -> 进入详情。
- 详情阅读：读取标题与元信息 -> 连续阅读正文 -> 浏览相关内容或返回列表。
- 异常兜底：遇到 404/空态 -> 理解当前状态 -> 返回首页或继续浏览可用内容。

## 页面与路由清单

- 首页：`/`，承担主题识别、重点内容展示、最新内容入口。
- 文章列表：文章聚合页或现有列表路由，承担批量扫描与分页/加载更多。
- 文章详情：文章详情路由，承担正文阅读、元信息、相关内容入口。
- 分类页：分类聚合路由，承担单一主题下的内容筛选。
- 标签页：标签聚合路由，承担更细粒度内容发现。
- 404 页：未命中路由，提供明确状态和返回路径。
- 空态页/空列表：无内容、无筛选结果时展示可恢复路径。

## 当前视觉问题清单

- 内容模块主次不清，首页与列表容易出现同等权重的信息堆叠。
- 区块线条、边框、分隔线数量偏多，造成页面碎片化。
- 卡片使用缺少场景边界，列表项、推荐块、正文容器容易互相抢层级。
- 留白节奏不稳定，区块间距、标题与正文间距、列表项间距缺少统一比例。
- 移动端可能出现内容拥挤、按钮/标签换行不稳、文字遮挡或横向滚动。

## 信息层级要求

- 首屏只保留一个最高视觉焦点，避免多个大标题、重卡片或强背景竞争。
- 标题层级控制在清晰的 H1/H2/H3 节奏，详情页正文优先于装饰元素。
- 元信息、分类、标签为辅助层级，颜色和字号弱于标题与正文。
- 列表页优先支持快速扫描：标题、摘要、时间/分类的顺序固定且密度适中。
- 404/空态应先说明状态，再提供单一主行动作，避免堆砌说明文案。

## 区块 / 线条 / 卡片 / 留白规则

- 区块：按阅读任务分组，不把每个小信息都做成独立区块。
- 线条：只用于强分组边界；同一区域内优先用留白和字体层级替代线条。
- 卡片：仅用于推荐内容、列表项或需要独立点击目标的内容；禁止卡片套卡片。
- 边框：统一粗细、颜色和圆角；弱边界优先使用浅色背景或间距。
- 留白：桌面端保证内容呼吸感，移动端保证标题、正文、标签、操作区不贴边。

## Tailwind 视觉验收

- 使用 Tailwind CSS v4 现有能力组织视觉规则，不引入新的视觉框架。
- 容器宽度、正文宽度、区块间距、列表间距应形成稳定 token，不逐页临时拼值。
- 颜色层级应包含正文、弱文本、边框、背景、强调色，不以单一灰阶或单色堆叠完成。
- 分隔线、卡片边框、阴影、圆角应有统一尺度，避免混用多套视觉语言。
- 响应式断点需覆盖桌面与移动端；移动端不得出现横向滚动、文字遮挡、按钮溢出。

## 桌面 / 移动端截图验收清单

- 桌面：首页首屏、首页下半屏、列表页、详情页首屏、详情正文段落、404、空态。
- 移动：首页首屏、列表页连续卡片/列表项、详情标题区、详情正文、404、空态。
- 截图检查：主视觉焦点唯一，区块边界清晰，线条不杂乱，列表可扫描，正文可连续阅读。
- 截图检查：移动端无横向滚动、无文字遮挡、无按钮溢出、无卡片过密。
- 截图检查：同类页面之间标题、间距、边框、标签、元信息样式一致。

## 开发顺序

1. 整理 `globals.css`：建立 Tailwind v4 `@theme` token、语义变量和共享 class。
2. 调整 `layout.tsx`：控制 shell、header、footer 的密度和边界。
3. 调整首页：明确最新重点文章、最近文章、分类/标签入口的主次。
4. 调整 `/articles`：统一单列列表样式。
5. 调整分类/标签页：复用列表结构，处理 active tag 和空状态。
6. 调整详情页：正文限宽、标题区层级、AI 摘要、相关内容。
7. 调整 404：简化结构和行动入口。
8. 回归 `/about`，确认共享样式未破坏阅读。
