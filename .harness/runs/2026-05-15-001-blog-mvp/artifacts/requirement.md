# 需求说明

## 背景

本 run 是个人博客 MVP 的第一个正式 run，用于把自然语言需求整理成可执行的工程边界，并确定最小可运行骨架方案。当前仓库已经具备 harness、monorepo 目录、`apps/web`、`apps/admin`、`apps/api`、`packages/types`、`packages/sdk`、`infra` 和基础 CI 目录，因此本轮重点是收敛需求、技术选型、项目结构与后续实现计划。

## 目标

- 面向读者：访问博客首页，查看已发布文章列表。
- 面向读者：进入文章详情页，查看标题、摘要、正文、发布时间和标签。
- 面向管理员：登录后台，并在登录失败或成功时看到明确反馈。
- 面向管理员：创建、编辑文章，支持保存草稿和发布。
- 面向后端：提供文章列表、文章详情、管理员登录、文章保存、文章发布的基础 API。
- 面向数据层：设计用户、文章、分类、标签及关联表的基础模型，并要求迁移可从空库执行。
- 面向工程：CI 至少覆盖 lint、typecheck、test 或 build 中的两个检查。

## 非目标

- 暂不做评论系统。
- 暂不做会员体系、多角色权限矩阵或 OAuth。
- 暂不做全文搜索服务。
- 暂不做复杂数据分析、访问统计或推荐系统。
- 暂不做富文本编辑器、图片上传、CDN、SSR/SSG 优化。
- 暂不做生产级部署自动化，本 MVP 只保留部署配置入口和发布说明。

## 用户故事

- 作为读者，我希望在首页看到已发布文章列表，以便快速了解博客最新内容。
- 作为读者，我希望打开文章详情，以便完整阅读文章正文和标签信息。
- 作为管理员，我希望能登录后台，以便进入文章管理工作区。
- 作为管理员，我希望能保存草稿，以便在未发布前持续编辑文章。
- 作为管理员，我希望能发布文章，以便文章出现在前台列表和详情页。
- 作为维护者，我希望 API、数据模型和 CI 有最小闭环，以便后续迭代有稳定基础。

## 验收标准

- [ ] 博客首页能展示已发布文章列表。
- [ ] 文章详情页能展示标题、摘要、正文、发布时间和标签。
- [ ] 后台登录失败和成功状态都有明确反馈。
- [ ] 后台文章表单支持保存草稿和发布。
- [ ] API 有基础错误处理，错误响应结构稳定。
- [ ] 数据库迁移可以从空库执行成功。
- [ ] CI 至少包含 lint、typecheck、test 或 build 中的两个检查。

## 影响范围

- [x] web
- [x] admin
- [x] api
- [x] db
- [x] infra
- [x] docs

## 测试要求

- API 测试覆盖文章列表、文章详情、登录失败、登录成功、保存草稿、发布文章。
- 前台测试覆盖列表渲染和详情展示。
- 后台测试覆盖登录状态反馈、草稿保存和发布流程。
- 数据库测试或脚本验证覆盖空库迁移。
- CI 至少执行两类检查，优先选择 `test` 和 `build`，如果引入 TypeScript 再加入 `typecheck`。

## 回滚方式

- 第一个 MVP 阶段以分支和 PR 为回滚边界。
- 业务代码回滚通过 Git revert 完成。
- 数据库迁移必须提供 down migration 或清晰的手动回滚步骤。
- 若本轮只产出计划，不涉及业务代码和数据库变更，则回滚方式为删除本 run 目录或 revert 对 `.harness/runs/2026-05-15-001-blog-mvp/` 的文档变更。

## 待确认问题

- 管理员认证第一版是否接受开发态固定账号密码，还是必须接入正式密码哈希与会话存储。
- 正文第一版使用 Markdown 纯文本、简化 textarea，还是需要富文本编辑体验。
- 数据库第一版是否确认使用 SQLite，本地文件库是否满足 MVP 部署预期。
- 前端是否继续保持原生 HTML/CSS/ESM，还是在下一轮升级为 Vite + React 或其他框架。
