# 001 个人博客 MVP

## 背景

需要为后续个人博客项目建立第一个可交付目标，覆盖 C 端浏览、管理端编辑、后端 API、数据库和基础部署。

## 目标

- 用户可以访问博客首页和文章详情页。
- 管理员可以登录后台。
- 管理员可以创建、编辑、发布文章。
- 后端提供文章相关 API。
- 数据库具备用户、文章、分类、标签的基础模型。
- CI 能运行基础检查。

## 非目标

- 暂不做评论系统。
- 暂不做会员体系。
- 暂不做全文搜索服务。
- 暂不做复杂数据分析。

## 验收标准

- [ ] 博客首页能展示已发布文章列表。
- [ ] 文章详情页能展示标题、摘要、正文、发布时间和标签。
- [ ] 后台登录失败和成功状态都有明确反馈。
- [ ] 后台文章表单支持保存草稿和发布。
- [ ] API 有基础错误处理。
- [ ] 数据库迁移可以从空库执行成功。
- [ ] CI 至少包含 lint、typecheck、test 或 build 中的两个检查。

## 影响范围

- [x] web
- [x] admin
- [x] api
- [x] db
- [x] infra
- [x] docs

## 测试要求

- 至少覆盖文章列表、文章详情、登录状态和文章保存流程。
- 如果暂未选择技术栈，先产出技术方案和项目初始化计划。

## 回滚方式

- 第一个 MVP 阶段以分支和 PR 为回滚边界。
- 数据库迁移必须提供 down migration 或明确的回滚步骤。
