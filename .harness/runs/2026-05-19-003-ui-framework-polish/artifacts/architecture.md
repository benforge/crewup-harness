# 架构和影响范围

## 影响范围

- [x] web
- [x] admin
- [ ] api
- [ ] db
- [ ] infra
- [x] docs

本轮为 UI 框架化与交互体验优化规划。开发阶段允许调整前端依赖、前端入口、页面布局、组件使用与样式组织；不修改 API 合约、数据库 schema、生产部署、密钥、CI/CD 与 `docs/product/`。

## 方案

- Admin 使用 Ant Design v5 作为后台 UI 基础，收敛布局、导航、表单、表格、消息反馈、确认弹窗、加载态、空态和错误态。
- Web C 端使用 Tailwind CSS v4 作为样式组织基础，优先保证核心阅读路径的容器、间距、响应式和交互状态一致。
- 本轮保持现有 API、数据库和部署边界不变，只在前端工程与 run artifacts 内实施。

## 模块边界

| 模块 | 本轮职责 | 边界 |
| --- | --- | --- |
| Admin 应用 | 接入 Ant Design v5，统一后台布局、导航、表单、表格、反馈、空状态、加载态、错误态 | 不改变业务数据流和接口语义；权限逻辑只做 UI 层闭环检查 |
| Web C 端 | 接入 Tailwind CSS v4，统一页面布局、响应式、核心阅读路径样式 | 不做品牌重制，不扩大为全站重构 |
| 共享配置 | 前端构建依赖、PostCSS/Tailwind 配置、主题 token 入口 | 不抽象跨端组件库，除非现有结构已有共享层 |
| 后端/API | 保持现有接口 | 仅当 UI 状态缺少必要字段时记录后续问题，不在本轮改约 |

## 路由与页面结构

| 区域 | 路由/页面 | 调整方式 |
| --- | --- | --- |
| Admin 公共入口 | 登录页、未授权页/跳转落点 | 保持现有路由语义，补齐登录失败、加载、跳转反馈 |
| Admin 受保护区域 | 后台框架页、仪表盘/首页、内容列表、内容编辑/表单、配置类页面 | 用 Ant Design `Layout` 承载外壳，导航保持现有信息架构；列表优先用 `Table`，表单优先用 `Form` |
| Web C 端 | 首页/列表、文章详情、分类/标签/搜索等核心阅读路径、基础导航与页脚 | 用 Tailwind utility 统一间距、排版、断点和容器宽度，优先保证桌面与移动端可读性 |
| 兼容检查 | 非核心页面 | 仅做样式冲突、布局破损与路由可达性检查 |

## 鉴权与权限边界

| 场景 | 边界策略 |
| --- | --- |
| 公开路由 | Web C 端阅读页、Admin 登录页保持公开 |
| 受保护路由 | Admin 后台主框架及其子页面继续走现有鉴权/守卫 |
| 登录后跳转 | 成功后回到原目标页或默认后台首页；UI 层补成功/失败提示 |
| 未授权处理 | 未登录跳登录；无权限显示现有未授权状态或统一错误提示，不绕过后端权限 |
| 会话失效 | API 返回未授权时清理前端会话状态并提示重新登录；避免静默失败 |

## 接口 / 数据依赖

- Ant Design v5：使用 `ConfigProvider` 统一主题 token，使用 `App` / `App.useApp()` 承载 `message`、`modal`、`notification` 上下文；后台基础组件优先选 `Layout`、`Menu`、`Form`、`Table`、`Modal`、`Spin`、`Empty`、`Result`。
- Tailwind CSS v4：开发阶段安装 `tailwindcss`、`@tailwindcss/postcss`、`postcss`；PostCSS 插件使用 `@tailwindcss/postcss`；Web 全局 CSS 使用 `@import "tailwindcss"`。
- 数据接口不新增、不变更。若页面需要更细的 loading/error/empty 判断，优先由前端基于现有响应状态封装。

## 实施步骤

1. 前端 agent 先确认 Admin/Web 应用结构、包管理器和现有入口文件。
2. 接入依赖与全局配置：Admin 增加 Ant Design provider；Web 增加 Tailwind/PostCSS 与全局 CSS 入口。
3. Admin 先迁移外壳、导航、表单、表格与核心操作反馈，确保鉴权流不回退。
4. Web 迁移核心阅读路径与响应式容器，保留现有内容结构。
5. Tester 覆盖构建、路由可达、Admin 鉴权/反馈、Web 桌面与移动端视觉回归。
6. Reviewer 聚焦依赖接入、样式冲突、权限边界和可回滚性。

## 风险与降级

- Ant Design CSS-in-JS 与现有样式冲突：先限制在 Admin 根节点与核心页面，必要时保留旧样式作为兜底。
- Tailwind utility 与现有 CSS 优先级冲突：优先迁移核心 C 端页面，非核心页面只做兼容检查。
- 依赖/lockfile 更新引入构建风险：单独提交依赖接入，构建失败可回滚 package 与配置。
- 交互反馈覆盖不完整：MVP 必须覆盖成功、失败、加载、确认、空状态；深色模式、多语言、完整无障碍增强不进入本轮。
- 页面清单未完全确认：按核心后台操作页和核心阅读路径先做，其他页面记录为后续迭代。
