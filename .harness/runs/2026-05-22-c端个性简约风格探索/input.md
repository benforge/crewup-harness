## 当前上下文

- 现有 C 端定位已经从通用博客收敛为“IT 工程师技术站 / 专业阅读体验”。
- 已有视觉基调偏克制、内容优先、低噪音，强调文章阅读、技术归档、照片墙作为次级视觉档案。
- 本次探索不应回到营销落地页、模板博客、花哨作品集或装饰堆叠，而是在简约基础上增加记忆点。

## 初步目标

- 让 C 端在第一屏和核心阅读路径上更有“个人风格”，但不牺牲专业、可信和可读性。
- 建立一个可持续的视觉语言：色彩、字体、留白、线条、图像/照片、代码感细节、微交互都服务同一个气质。
- 把“个性但简约”转成可截图验收的标准，而不是泛泛地调颜色、加装饰。

## 推荐初选

建议先以“工程手札”为主方向，融合少量“冷静编辑部”的排版秩序：

- 个性来源：作者身份、工程笔记感、真实工作痕迹、代码阅读细节。
- 简约边界：首屏只保留一个焦点，区块少，线条少，装饰只用于提示结构。
- 视觉关键词：克制、纸面、标注、技术笔记、可读、可信、有一点温度。

## 待确认问题

- 你希望 C 端更像“个人工程师手记”，还是更像“技术杂志/编辑部”？
- 个性更想来自哪里：颜色、字体、照片、代码感细节、文案语气，还是布局结构？
- 是否接受首页保留简短英文技术感标题，还是整体更偏中文表达？
- 照片墙要继续做轻量次级入口，还是希望它参与首页个性塑造？
- 你有没有不喜欢的风格：例如太暗、太日系、太赛博、太企业官网、太 Notion、太营销页？

## 后续进入实现前需要补齐

- 明确主风格方向与禁用风格。
- 明确影响页面范围：至少首页、文章列表、文章详情、关于页、照片入口、404/空态。
- 明确截图验收视口：桌面和移动端各覆盖首页、列表、详情。
- 明确是否只做 C 端，不碰 Admin。
