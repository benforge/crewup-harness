# Frontend Rules

## Principles

- Read existing UI patterns before implementation.
- Keep layout responsive across desktop and mobile when the requirement touches user-visible UI.
- Prefer accessible controls, clear states, and stable dimensions.
- Do not introduce a frontend framework unless the project already uses it or the architecture plan requires it.

## Verification

- Run the available build command when present.
- For browser-facing work, verify the page is non-blank and core interactions work.
